Typst presentations with motion

Glissade is an early-stage toolkit for adding animation state to Typst slides, previewing them in the browser, and exporting a portable HTML presentation.

#slide(autoplay: true)[
  = Hello Glissade

  #init(width: 3cm)
  #animate(width: 8cm)
  #animate(width: 3cm)
  #cut(loop: true)

  #context [
    #rect(
      width: a("width"),
      height: 1cm,
      fill: blue,
    )
  ]
  #finish()
]

01 / What it does

A practical path from Typst source to browser presentation.

Timeline as code

Animate values, coordinate blocks, define cuts, and reuse ordinary Typst definitions across every slide.

Vector-native motion

Morph formulas and SVG geometry in the browser without giving up sharp, resolution-independent output.

HTML export

Preview changes live, navigate animation boundaries, inspect a slide overview, and export one self-contained HTML file.

02 / Workflow

Stay in Typst.
Test in the browser.

  1. 1
    Describe

    Build slides with familiar Typst markup and Glissade’s small animation API.

  2. 2
    Preview

    Use the live editor to scrub frames and rebuild only the slide you are working on.

  3. 3
    Export

    Package compressed SVG frames and the browser runtime into a single file. Try the example deck.

03 / Status

Early version, open design.

Glissade is still experimental, so the Typst API and export format may change. It uses Kino as the core animation starting point and explores how that proof of concept can fit a browser-focused presentation workflow.
