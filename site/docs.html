Quick documentation

From zero to an animated deck.

Glissade keeps animation state in Typst and renders playback in the browser. This guide covers the shortest useful workflow; the PDF manual contains the full API reference.

Install

Import the package from Typst Universe:

#import "@preview/glissade:0.1.0": *

Create your first slide

#set page(width: 128mm, height: 96mm)
#show: deck.with(fps: 12)

#slide(title: "A moving value")[
  #init(x: 1cm)
  #animate(x: 8cm, duration: 1.2)

  #context {
    circle(radius: a("x") / 8, fill: blue)
  }
  #finish()
]

init establishes a value, animate creates a timeline block, and a reads the value for the current frame.

Preview and export

# Live editor
python3 bin/glissade.py --root . slides.typ dev --fps 12

# Self-contained HTML presentation
python3 bin/glissade.py --root . slides.typ html --fps 24

The HTML exporter embeds the runtime and gzip-compressed SVG frames. Use --no-embed-frames while inspecting output or serving frames separately. The GitHub Pages site includes an exported example deck generated from examples/slides.typ at 25 fps.

Morph formulas and shapes

#init(formula: $part(a) + part(b)$)
#animate(formula: $part(a) / part(b)$)

#context {
  glissade-morph("formula")
}

Matching part markers move between keyframes. Unmatched fragments fade, and ordinary SVG geometry can be morphed or drawn into view.
