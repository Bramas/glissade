Small Typst examples, real rendered slides.

Each feature below has a focused source snippet on the left and a rendered Glissade HTML slide on the right. The previews are exported from the matching files in site/examples/, so they exercise the same browser runtime used by actual presentations.

Getting started

A Glissade deck is an ordinary Typst document with a deck show rule and one or more slide blocks. Inside a slide, animation state is declared with primitives such as init, animate, and cut, then read with a("name") inside context.

#import "@preview/glissade:0.1.0": *
#set page(width: 128mm, height: 96mm)
#show: deck.with(fps: 12)

#slide(title: "A first animation")[
  #init(width: 1cm)
  #animate(width: 8cm)
  #wait(duration: .45)
  #cut(loop: true)

  #context [
    #rect(width: a("width"), height: 1cm)
  ]
  #finish()
]

Animate a value

init gives a variable its starting value. animate creates the next timeline block and interpolates that variable to a new value. Use a("width") from a Typst context to read the value for the current frame.

animate-width.typvalue state
#init(width: 42pt)
#animate(width: 220pt, duration: 1.1)
#wait(duration: .45)
#cut(loop: true)

#context [
  #rect(
    width: a("width"),
    height: 34pt,
    radius: 5pt,
    fill: accent,
  )
]

Coordinate variables with meanwhile

meanwhile adds another variable animation to the current block. It is useful when several visual properties should change together: position and opacity, scale and color, or multiple diagram coordinates.

meanwhile.typsame block
#init(x: 20pt, opacity: 20%)
#animate(x: 210pt, duration: 1.2)
#meanwhile(opacity: 100%, duration: 1.2)
#wait(duration: .45)
#cut(loop: true)

#context [
  #place(dx: a("x"), dy: 32pt)[
    #circle(
      radius: 23pt,
      fill: accent.transparentize(100% - a("opacity")),
    )
  ]
]

Sequence steps with then, wait, and cut

then appends an animation after the current block, wait holds state for a duration, and cut creates a playback boundary. In the browser, advancing moves through cuts before moving to the next slide.

then-wait-cut.typpresenter boundary
#init(step: 0)
#animate(step: 1, duration: .45)
#then(step: 2, duration: .45)
#wait(duration: .35)
#then(step: 3, duration: .45)
#wait(duration: .55)
#cut(loop: true)

#context [
  // Render each box as active when step reaches it.
]

Loop a segment

Pass loop: true to cut when a segment should keep playing until the presenter advances. This is useful for ambient motion on title slides, waiting screens, or diagrams that should stay alive while you talk.

loop.typlooping cut
#init(width: 48pt)
#animate(width: 230pt, duration: .9)
#animate(width: 48pt, duration: .9)
#wait(duration: .35)
#cut(loop: true)

#context [
  #rect(
    width: a("width"),
    height: 40pt,
    radius: 20pt,
    fill: rgb("#19a974"),
  )
]

Pick a transition function

Animation primitives accept transition. Built-ins include linear, smooth, quad, cubic, quart, sin, and circ. You can also provide a custom function from [0, 1] to [0, 1].

transitions.typcurves
#init(linear-x: 0pt, smooth-x: 0pt, sin-x: 0pt)
#animate(linear-x: 200pt, duration: 1.2, transition: linear)
#meanwhile(smooth-x: 200pt, duration: 1.2, transition: smooth)
#meanwhile(sin-x: 200pt, duration: 1.2, transition: sin)
#wait(duration: .45)
#cut(loop: true)

#context [
  #row("linear", 28pt, a("linear-x"), blue)
  #row("smooth", 78pt, a("smooth-x"), green)
  #row("sin", 128pt, a("sin-x"), purple)
]

Morph formulas by marking parts

Wrap a formula in formula(...), mark the pieces that should be tracked with part, then render the state with glissade-morph. Matching parts move between frames, while unmatched pieces can fade in or out.

formula-morph.typsemantic SVG
#init(eq: formula($part(a) part(=) part(b)$))
#animate(
  eq: formula($part(a) / part(c) part(=) part(b) / part(c)$),
  duration: 1.1,
)
#wait(duration: .45)
#cut(loop: true)

#align(center)[
  #context [
    #text(size: 34pt)[#glissade-morph("eq")]
  ]
]

Create content with draw-border-then-fill

create introduces content from an empty state. By default it attaches the draw-border-then-fill morph effect, so the browser runtime can draw the shape before filling it.

create-reveal.typcontent reveal
#create(card: [
  #rect(
    width: 230pt,
    height: 92pt,
    radius: 8pt,
    fill: rgb("#d8e9df"),
    stroke: 1pt + rgb("#246b61"),
  )[New idea]
], duration: 1.5)
#wait(duration: .45)
#cut(loop: true)

#align(center)[
  #context [#glissade-morph("card")]
]

Use slide metadata

slide accepts an explicit id, a browser overview title, autoplay, and a local fps. Use slide-number() when the rendered content should display its position in the deck.

slide-metadata.typslide options
#slide(
  id: "overview-title",
  title: "Overview title",
  autoplay: true,
  fps: 16,
)[
  #init(width: 70pt)
  #animate(width: 210pt, duration: .9)
  #wait(duration: .45)
  #cut(loop: true)
  // ...
  slide #slide-number()
]

Export and preview

The command-line bridge compiles Typst, reads Glissade metadata, extracts SVG frames, and packages the browser runtime. Use dev while authoring and html when you want a portable presentation.

# Live editor
python3 bin/glissade.py --root . slides.typ dev --fps 12

# Self-contained HTML presentation
python3 bin/glissade.py --root . slides.typ html --fps 24 --title "My talk"

The full PDF manual goes deeper into supported value types, generated metadata, export flags, and the lower-level Kino-compatible outputs.
